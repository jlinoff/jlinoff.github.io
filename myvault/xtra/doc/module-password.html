(static) makePasswordEntryWithId(eid, cls, placeholder, value) → {element}

Create the password input DOM element with the buttons to show/hide button, generate cryptic password button generate the memorable password button, copy the password button and the password length display element with a user specified input id and class to make it easier to directly access and manipulate the input elements.

This is used to make custom password inputs for each record which is why the getter an setter functions are not needed.

Note that this code need to be DRYed up in the future because it replicates code in makePasswordEntry().

Parameters:
Name Type Description
eid string

The input element id.
cls string

The input element class.
placeholder string

The placeholder text in the input element.
value string

The initial password value. Can be overwritten interactively.

The password options object. It defines the alphabet, the minimum length (minlen) and maximum length (maxlen) of the password that is to be generated. The default minimum length is 15 and the maximum length is 31.

(inner) _generateMemorablePassword() → {string}

Generate a human readable (memorable) password between 15 and 31 characters where each word is separated by a forward slash.

The decision to not parameterize the word size, password size and separator was deliberate to keep things simple. In the future it might make sense to make those parameters dynamic by adding them to the common parameters.
